Provides the classes and methods necessary for authenticating with Identity4J

Before using the Identity4J library, you need a few configuration items. You will need credentials for your account to connect to the service. We can supply you with the proper credentials which require a public/private key pair and certificate or you can generate your own public/private keypair and certificate signing request for us to sign. To have us provide you the necessary credentials including private key, please contact us at info@ufp.com.

To generate your own private key and CSR you can use the popular OpenSSL program, which is already installed on most Unix systems.

To generate a private/public key pair:

openssl req -new -newkey rsa:1024 -keyout example.com.key.pem -keyform pem -out example.com.req.pem -outform pem

Which will generate a new 1024 bit key and write it to example.com.key.pem and a CSR in example.com.req.pem. Enter a good, strong password for your key when openssl prompts you:

Enter PEM pass phrase:
Verifying - Enter PEM pass phrase:

The necessary elements of the CSR are in blue, the optional ones in green:

Country Name (2 letter code) [US]:
State or Province Name (full name) [Some-State]:
Locality Name (eg, city) []:
Organization Name (eg, company) [Example, Inc.]:
Organizational Unit Name (eg, section) []:
Common Name (eg, YOUR name) [example.com]:
Email Address []:

Send the resulting CSR to us at info@ufp.com and we will generate a PKCS#12 format file for you.

Now you must configure the provider with your newly created PKCS#12 file and the truststore (also available from info@ufp.com).
